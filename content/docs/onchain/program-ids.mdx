  this page with cluster + slot metadata.

## Responsibilities

| Program          | Owns                                                                                   |
| ---------------- | -------------------------------------------------------------------------------------- |
| `xvault_vault`   | Vault state, deposits, withdrawals, rebalance bookkeeping, fee accrual, pause controls |
| `xvault_oracle`  | Keeper-pushed NAV snapshots with per-leg multiplier-aware entries                      |
| `xvault_rewards` | `$VLT` staking, lockup tracking, epoch Merkle roots, claim proofs                      |

See [Architecture](/docs/protocol/architecture) for the full component map and data flow, and [Reference → Instructions](/docs/onchain/instructions) for the per-ix authority and effect table.

## Notes for integrators

- **Verify** deployment status before wiring production clients. The canonical status lives in this page and the release notes.
- **IDL + SDK** artifacts are the canonical source for instruction layouts. Consume `packages/sdk` rather than hand-rolling account parsing.
- **No mainnet assumption.** Do not assume these IDs imply mainnet availability until the rollout is complete.
- **Squads authority.** All admin-authority ix require a Squads v4 multisig signer; integrating clients should expect multi-sig batching rather than single-wallet submission for those paths.
